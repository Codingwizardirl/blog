Fix for external monitors with varying DPI.

Posted by Pavel Georgiev on 2016-09-07

If you’ve read my last blog you know that for some time now I have been using Linux on my machine for software development.
Ok, now comes the real question. If I am not using an external monitor, am I really a programmer?

Disclaimer: This setup is sadly not mine but of a fellow reddit user from /r/battlestations

So I purchased an external monitor and wasn’t extremely excited to find out that once I plugged it in, it used the scale factor of my laptop screen, which for a 3840x2160 screen is x2. As you might have guessed by now, everything looked huge. So I began digging in the world of esoteric, also known as the Internet. This lead me to solutions for old Ubuntu versions, which sadly did not work for me on Ubuntu 16.04 and gnome-shell 3.18.5. So if you are running into the same problem I might have a fix for you.

We will be using xrandr to fix the issue.

First run the this command in your terminal so you know what resolutions your screens are running on.

In my case I have my laptop monitor(eDP1) running on 3840x2160 and my external monitor(HDM1) running on 1680x1050.
So note those numbers down and once you plug them in the formula I am about to give you, you should be good and running.
Do not worry if you don’t get it right from the first try, the changes are not permanent and upon restart everything will be back to “normal”. This is a blessing and a curse but I will get into that in a bit.
